# Fankui

当用户提到 `/fankui`、要求处理反馈文件、分析报错日志、查看问题截图，或明确要求基于 `问题反馈/` 修复问题时，使用这个技能。

## 触发条件

- 用户输入 `/fankui`
- 用户说“看一下问题反馈”“按反馈修一下”“分析截图/日志”
- 用户要求先扫描 `问题反馈/` 文件夹

## 工作流程

1. 确认项目根目录存在 `问题反馈/`
2. 扫描 `问题反馈/` 及子目录，优先处理最近修改的文件
3. 读取文本文件，提取报错、日志、用户描述、复现线索
4. 如果存在图片，客观描述画面中的界面异常、报错信息或布局问题
5. 输出统一分析结果：
   - 文件清单
   - 问题总结
   - 用户指令解析
   - 最小改动优化方案
   - 下一步确认
6. 在修改前先读取目标代码，理解现有实现
7. 修改完成后说明变更原因，并把重要结果回写到 `CLAUDE_PROJECT_MEMORY.md`

## 执行原则

- 优先级：修复 bug > 性能 > 可读性 > UX > 新功能
- 默认采用最小可行改动，不做与反馈无关的重构
- 不要先问用户“文件夹在哪”，直接读取仓库内 `问题反馈/`
- 图片描述必须客观，不猜测用户意图
- 所有对用户沟通使用中文
