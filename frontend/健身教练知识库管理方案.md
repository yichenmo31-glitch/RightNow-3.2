## 📝 文件命名规范

### 通用规则
- 使用中文命名（便于阅读和检索）
- 格式：`类型-主题-子主题.md`
- 避免特殊字符：`/ \ : * ? " < > |`
- 日期格式：`YYYY-MM-DD`

### 示例
```
✅ 正确：
- 训练计划-增肌-初级.md
- 营养指南-减脂期饮食.md
- 视频笔记-2024-01-15-深蹲技巧.md

❌ 错误：
- plan.md（不明确）
- 增肌训练计划（2024/1/15）.md（包含特殊字符）
```

## 🔄 内容处理流程

### 阶段1：原始资料收集
```
原始资料/
├── books-pdf/          # PDF 扫描版
├── books-epub/         # 电子书
├── videos-raw/         # 视频文件
└── articles/           # 文章链接
```

### 阶段2：内容提取与转换
1. **PDF → Markdown**
   - 使用 OCR 工具（如 Mathpix）
   - 人工校对关键章节

2. **视频 → 文字**
   - 使用语音转文字（讯飞、剪映）
   - 提取关键帧图片
   - 整理成结构化笔记

3. **网页 → Markdown**
   - 使用 Defuddle 或 Jina Reader
   - 保留原始链接

### 阶段3：内容标准化
1. 添加 YAML 元数据
2. 统一格式和术语
3. 提取核心概念
4. 添加 RAG 检索关键词

### 阶段4：质量审核
- [ ] 内容准确性检查
- [ ] 格式规范性检查
- [ ] 元数据完整性检查
- [ ] 关键词覆盖度检查

## 🤖 RAG 系统集成准备

### 文本分块策略
```python
# 推荐配置
chunk_size = 800          # 字符数
chunk_overlap = 200       # 重叠字符数
separator = "\n## "       # 按二级标题分块
```

### 向量化配置
```python
# 推荐模型
embedding_model = "text-embedding-3-large"  # OpenAI
# 或
embedding_model = "bge-large-zh-v1.5"       # 本地部署
```

### 元数据字段（用于过滤）
- `category`: 知识类别
- `difficulty`: 难度等级
- `target_audience`: 目标人群
- `tags`: 标签列表
- `source_credibility`: 来源可信度

## 📊 质量控制标准

### 内容质量评分（1-10分）
- **9-10分**：权威来源 + 完整结构 + 实践案例
- **7-8分**：可靠来源 + 清晰结构
- **5-6分**：一般来源 + 基本结构
- **<5分**：需要补充或重新整理

## 💡 关键建议

1. **先做减法再做加法**
   - 不要一次性整理所有内容
   - 先整理最核心的 20% 内容（覆盖 80% 需求）

2. **保持一致性**
   - 统一术语（如"肌肥大" vs "增肌"）
   - 统一格式（标题层级、列表样式）

3. **注重可检索性**
   - 每个文档至少 5 个关键词
   - 关键词要覆盖：学术术语 + 通俗说法

4. **定期更新**
   - 每月更新 update-log.md
   - 每季度审核内容时效性
