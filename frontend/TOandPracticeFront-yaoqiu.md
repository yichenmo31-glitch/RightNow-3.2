# TODO + 训练记录前端实现需求文档

## 📋 概述

后端已完成 TODO + 训练记录功能的全部实现，前端需要配套 UI。核心流程：用户查看 TODO → 点击训练任务 → 记录训练 → 查看 AI 反馈 → 转发社区。

**实现状态**：✅ 已完成基础功能实现，需要 UI/UX 优化

---

## 🎯 核心功能流程

```
TodoList (查看待办)
    ↓ 点击训练任务
TrainingLog (记录训练)
    ↓ 提交后获得 AI 反馈
    ↓ 点击"转发到社区"
CommunityShare (编辑分享内容)
    ↓ 发布
Community (社区展示)
```

## 📱 设计规范

### 配色方案
- **背景色**：`#030303`（深黑）
- **主色**：`#B8FF00`（荧光绿）
- **卡片背景**：`#1a1a1a`（浅黑）
- **文字**：`#FFFFFF`（白色）、`#999999`（灰色）

### 字体规范
- **标题**：`text-2xl font-bold`（24px 粗体）
- **正文**：`text-base`（16px 常规）
- **辅助文字**：`text-sm text-gray-400`（14px 灰色）

### 圆角规范
- **卡片**：`rounded-lg`（8px）
- **按钮**：`rounded-lg`（8px）
- **标签**：`rounded-full`（完全圆角）

### 间距规范
- **卡片间距**：`mb-3`（12px）
- **内边距**：`p-4`（16px）
- **按钮高度**：`py-3`（上下 12px）

### 按钮规范
- **主按钮**：绿色背景 + 黑色文字 + 粗体
- **次按钮**：灰色背景 + 白色文字
- **禁用状态**：降低透明度 + 禁用点击

## 📝 注意事项

### 开发注意
1. **加载状态必须显示**：AI 调用需要 2-3 秒
2. **错误处理必须友好**：API 失败时显示友好提示
3. **空状态必须引导**：TODO 列表为空时显示引导文案
4. **语音兼容性检测**：不支持时隐藏语音按钮
5. **图片上传使用现有 API**：`uploadApi.upload()`

### 设计注意
1. **移动端优先**：所有设计基于移动端
2. **安全区域适配**：考虑刘海屏和底部横条
3. **深色主题一致性**：所有页面保持深色风格
4. **主色使用克制**：荧光绿仅用于强调和主按钮
