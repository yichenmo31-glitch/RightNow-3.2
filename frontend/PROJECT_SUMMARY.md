# RightNow - 项目总结与分析文档

## 一、 项目背景与概述
**RightNow** 是一款主打“AI 驱动”与“3D 视觉反馈”的现代化智能健身记录与指导 Web 应用（PWA 优先设计）。它突破了传统健身 App 枯燥的数据记录模式，通过生成个人专属的 3D 进化模型、提供基于 AI 的饮食分析与训练规划，以及极具未来感和沉浸感的 UI 设计（玻璃拟态、霓虹高亮、流畅动画），旨在为用户提供高度个性化、可视化的身材管理体验。

## 二、 核心技术栈
项目采用了现代化、轻量且高性能的前端技术栈：
- **核心框架**: React 19 + TypeScript + Vite构架，极致的开发与编译体验。
- **UI & 样式**: 
  - Tailwind CSS：提供高度定制化的原子化 CSS 体系，构建玻璃拟态（Glassmorphism）和暗黑主题（Dark Mode）的未来感视觉风格。
  - Material Icons Round：作为项目全局标配的高质感图标库。
- **3D 渲染与可视化**:
  - `three` & `@react-three/fiber` & `@react-three/drei`：用于构建“进化引擎（Evolution Engine）”，渲染用户的 3D 身体状态和数字孪生动画。
- **数据可视化**: 
  - `recharts`：用于渲染美观的“体重与身材数据看板”（折线图等）。
- **状态管理**: 
  - 依赖 React 自身的 Local State 与通过外层组件（如 `App.tsx`）提升至全局的 Shared State（如 `weightData`、`customPhotos` 等）进行状态流转。

## 三、 系统功能模块

### 1. 引导与 3D 建模系统 (Onboarding & Evolution Engine)
- **精准身材录入**：包含性别、身高、体重、年龄数据的沉浸式录入。
- **理想态定制**：男生面板提供“薄肌型”、“力量型”、“健美型”及自定义目标（支持上传图片）；女生面板提供“漫画型”、“力量型”、“健美型”及自定义目标。
- **视觉扫描打卡**：支持调用系统相机或相册上传全身照。
- **3D 进化展示**：提交数据后进入 `EvolutionEngine.tsx`，展示基于当前状态到理想状态的 3D 模型生成演算与视觉渲染。

### 2. 智能工作台与数据看板 (Dashboard & Stats)
- **首页 Dashboard**：聚合展示今日计划、运动连胜、消耗数据，作为进入各个子模块的中枢入口。
- **数据看板 DataDashboard**：
  - 基于 Recharts 呈现体重趋势折线图。
  - **历史形态 Evolution Record**：展示从过去到现在的身材照片墙时间线流，支持从本页面直接点击右上角进入上传流程，并在**进化图库（EvolutionGallery）**中以网格归档展示。
  - **记录体重 Weight Record**：提供直观的体重与腰围记录表单。

### 3. 饮食与营养追踪 (Diet Log)
- **可视化饮食展现**：将一天的饮食记录以极具食欲的大图卡片展示，右上角附带悬浮的卡路里与三大宏量营养素（碳、蛋、脂）数据标签。
- **Telegram 风格动作面板**：
  - 点击底部的“加号”菜单，可呼出“纯文字记录”或“拍照记录”选项。
  - 选择**拍照**：从底部弹出仿 TG 的上拉相册抽屉式菜单，支持实时相机、文件与相册网格极速预览读取。
  - 选择**文字**：全屏弹出文本输入毛玻璃 Modal，收集食物名称和描述后，“**AI 分析**”按钮结合后台大模型能力自动反推估算卡路里。

### 4. 运动极速记录 (Action Center)
- **中央加号唤出**：位于全局 Navigation 中央的独立操作台。
- **语音/文本记训**：支持通过点击麦克风唤起类似语音助手的录音动效，自动转写健身记录。
- **视觉打卡**：拍完照后不仅在此处记录，打通的全局共享状态会自动同步将其放入“进化图库 (EvolutionGallery)”，形成连贯反馈。

### 5. 专属全天候 AI 教练 (Floating Advisor & AIChat)
- **无处不在的悬浮球**：除了在完全沉浸的拍照/3D/表单页面，其余业务页面右下角均有 AI 形象球常驻。
- **未读提示与智能介入**：当用户完成初始组建（如 Onboarding 结束），AI 会自动冒红点，提供“接下来的专属计划介绍”。
- **全屏流式对话框**：进入真正的 AI 聊天页面后，可进行健身疑问解答、计划调整、饮食评测，UI 呈现聊天气泡。

### 6. 打卡与分享链路 (Check-In Flow)
包含完善的打卡动线：`CheckInType`（选择打卡种类） -> `CheckInBody`（录入当日体感与数据） -> `CheckInSuccess`（生成优美的打卡海报） -> `CheckInShare`（调取原生系统分享到社交媒体）。

## 五、 技术开发与体验亮点
1. **统一的高级视觉语言**：采用黑/深灰色调为主体，配以 `#B8FF00` （霓虹绿）等鲜明的高饱和度关键色调作为点缀，运用丰富的 `backdrop-blur` 和 `border-white/10` 以及 `box-shadow` 发光特性来实现毛玻璃质感与科技感。
2. **状态闭环与跨试图数据互通**：比如在 `ActionCenter` 拍照记录的图片，通过 `App.tsx` 中的全局 State `customPhotos` 进行存储，并实时下发到 `EvolutionRecord` 和 `EvolutionGallery`，让用户的每一个数据上传动作都能在应用各处得到呼应反馈。
3. **细节交互的持续调优**：比如 `DietLog` 中的 Telegram 风格底部上拉列表、材质防翻译导致渲染变形的技术处理 (`notranslate` 类名)、沉浸式 Modal 的毛玻璃遮罩以及无状态或暂无数据时的兜底页面等，都是高度侧重 C 端质感的交互实践。
