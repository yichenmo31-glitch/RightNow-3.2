# 代码审查规范

## 审查清单

### 功能性
- [ ] 功能是否符合需求
- [ ] 边界情况是否处理
- [ ] 错误处理是否完善

### 代码质量
- [ ] 代码是否清晰易读
- [ ] 命名是否规范
- [ ] 是否有重复代码
- [ ] 是否过度设计

### 性能
- [ ] 是否有性能问题
- [ ] 数据库查询是否优化
- [ ] 是否有内存泄漏

### 安全
- [ ] 是否有安全漏洞
- [ ] 输入是否验证
- [ ] 敏感数据是否加密

### 文档
- [ ] 是否更新相关文档
- [ ] 复杂逻辑是否有注释

## 审查流程

1. 自测通过后提交PR
2. 指定审查者
3. 审查者检查代码
4. 提出修改意见
5. 作者修改代码
6. 再次审查
7. 批准合并

## 审查意见格式

```markdown
**问题**: 描述问题
**建议**: 修改建议
**优先级**: 高/中/低
```
