# Fankui

当用户要基于反馈文件、日志、截图来定位和修复问题时，使用这个技能。

## 触发信号

- 用户输入 `/fankui`
- 用户要求分析 `问题反馈/`
- 用户明确说按反馈修 bug、看截图、看日志

## 执行顺序

1. 扫描 `问题反馈/` 目录及子目录，优先看最新文件
2. 读取文本反馈，提取报错、日志、复现线索
3. 如果有图片，按界面问题或报错截图进行客观描述
4. 汇总问题并给出最小改动方案
5. 先读目标代码，再实施修改
6. 修改后说明变更原因，并更新共享记忆中的“最近变更”或“风险/阻塞”

## 固定输出结构

1. 文件清单
2. 问题总结
3. 用户指令解析
4. 优化方案
5. 下一步确认
